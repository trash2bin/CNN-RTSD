# CNN для классификации дорожных знаков (RTSD)

Свёрточная нейронная сеть для классификации изображений дорожных знаков из датасета [RTSD (Russian Traffic Sign Dataset)](https://huggingface.co/datasets/eleldar/rtsd_cleaned). Проект реализован на PyTorch с нуля без использования предобученных моделей. Помимо самой классификации, исследованы вычислительная сложность модели, её устойчивость к различным видам искажений и интерпретируемость предсказаний через Grad-CAM и Saliency Maps.

## Описание задачи

Распознавание дорожных знаков — одна из ключевых задач компьютерного зрения, применяемая в системах помощи водителю (ADAS) и автономных транспортных средствах. Задача осложняется разнообразием условий съёмки: разное освещение, частичное перекрытие знаков, смазанность изображения и атмосферные помехи.

**Цели проекта:**

- разработать и обучить CNN с нуля на датасете RTSD
- провести эмпирическую верификацию теоретической вычислительной сложности и потребления памяти
- исследовать устойчивость модели к различным видам искажений тестовых изображений
- применить методы интерпретируемости (Grad-CAM, Saliency Map) для анализа ошибок

## Архитектура модели

Трёхблочная CNN с постепенным увеличением числа каналов и уменьшением пространственного разрешения. Входное изображение: **3 × 48 × 48** (RGB).

```python
# Блок 1: 3 → 32 каналов, 48×48 → 24×24
block1 = Sequential(
    Conv2d(3, 32, kernel_size=3, padding=1),
    BatchNorm2d(32), ReLU(),
    Conv2d(32, 32, kernel_size=3, padding=1),
    BatchNorm2d(32), ReLU(),
    MaxPool2d(2, 2), Dropout2d(0.25)
)
# Блок 2: 32 → 64 каналов, 24×24 → 12×12
block2 = Sequential(
    Conv2d(32, 64, kernel_size=3, padding=1),
    BatchNorm2d(64), ReLU(),
    Conv2d(64, 64, kernel_size=3, padding=1),
    BatchNorm2d(64), ReLU(),
    MaxPool2d(2, 2), Dropout2d(0.25)
)
# Блок 3: 64 → 128 каналов, 12×12 → 6×6
block3 = Sequential(
    Conv2d(64, 128, kernel_size=3, padding=1),
    BatchNorm2d(128), ReLU(),
    Conv2d(128, 128, kernel_size=3, padding=1),
    BatchNorm2d(128), ReLU(),
    MaxPool2d(2, 2), Dropout2d(0.25)
)
gap = AdaptiveAvgPool2d(1)
classifier = Sequential(
    Flatten(),
    Linear(128, 256), ReLU(), Dropout(0.5),
    Linear(256, num_classes)
)
```

**Ключевые решения:**

- **Global Average Pooling** вместо Flatten значительно сокращает число параметров классификатора и добавляет инвариантность к пространственным сдвигам
- **BatchNorm** после каждой свёртки стабилизирует обучение
- **Dropout2d(0.25)** в блоках и **Dropout(0.5)** в классификаторе регуляризация против переобучения
- Три блока выбраны как компромисс: большее число привело бы к чрезмерному уменьшению разрешения до GAP, меньшее к недостаточной глубине иерархии признаков

**Параметры модели:** ~4,34 млн  
**Размер весов (float32):** ~16,55 MiB


## Датасет и подготовка данных

**Датасет:** [eleldar/rtsd_cleaned](https://huggingface.co/datasets/eleldar/rtsd_cleaned) — реальные снимки дорожных знаков РФ с бортовых видеорегистраторов.

**Разбивка:** датасет содержит одну общую выборку, которая делится в соотношении **90/10** с фиксированным random seed для воспроизводимости.

**Предобработка:**
- приведение изображений к размеру **48 × 48**
- нормализация со статистиками ImageNet (mean/std)
- переотображение числовых меток классов в плотный диапазон [0, N)

**Аугментации (только для train):**
- случайные повороты и горизонтальное отражение
- изменение яркости, контраста, насыщенности
- гауссовский шум и случайное размытие
- случайное закрытие части изображения (Random Erasing)

**DataLoader:** batch size = **64**


## Обучение

| Компонент | Значение |
|-----------|----------|
| Функция потерь | CrossEntropyLoss |
| Оптимизатор | Adam, $lr = 1×10⁻³$ |
| Планировщик | Cosine Annealing ($T_{max}$ = 50, $η_{min} = 1×10⁻⁶$) |
| Регуляризация | BatchNorm + Dropout2d(0.25) + Dropout(0.5) |

**Cosine Annealing** позволяет делать крупные шаги в начале обучения и плавно досходиться к минимуму в конце, снижая риск застревания в локальных минимумах.

**CrossEntropyLoss** в реализации PyTorch совмещает Softmax и вычисление кросс-энтропии в одном численно устойчивом шаге, исключая проблему исчезающего градиента при работе с логитами.

## Исследование устойчивости

Тестовая выборка дополнялась тремя видами искажений для оценки робастности модели:

| Тип искажения | Уровни | Что моделирует |
|--------------|--------|----------------|
| Гауссовский шум | σ = 0.05 / 0.10 / 0.20 | Съёмка при низкой освещённости |
| Размытие | радиус = 1 / 3 / 5 | Движение камеры, тряска |
| Окклюзия | 10% / 30% / 50% площади | Частичное перекрытие знака |

По каждому типу и уровню строится «профиль устойчивости» — зависимость Accuracy от интенсивности искажения. Дополнительно сравниваются модели, обученные **без аугментаций** и **с аугментациями**, что позволяет количественно оценить их практическую ценность.


## Визуализация ошибок

Два метода интерпретируемости применяются к примерам, на которых модель допускает ошибки:

**Grad-CAM (Gradient-weighted Class Activation Mapping)** — вычисляет градиенты скора целевого класса относительно карт признаков последнего свёрточного слоя, взвешенно суммирует их и строит тепловую карту значимых регионов входного изображения. Применяется к последнему Conv-слою, поскольку он содержит семантически насыщенные признаки.

**Saliency Map** — вычисляет градиент скора предсказанного класса по каждому пикселю входного изображения. Даёт детальную попиксельную картину внимания модели, однако более зашумлённую по сравнению с Grad-CAM.

Совместный анализ позволяет выявлять систематические паттерны неверной классификации: смещение внимания на фон, блики, элементы разметки вместо самого знака.


## Результаты тестирования

На стандартных тестовых примерах (знак занимает значительную часть кадра) модель уверенно классифицирует знаки. При работе с изображениями, где знак является лишь небольшой частью фото, качество предсказания снижается — это ограничение обусловлено небольшим входным разрешением (48 × 48) и особенностями обучающей выборки.
